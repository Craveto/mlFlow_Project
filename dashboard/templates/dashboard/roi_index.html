How It Works

Minimal assumptions, useful output

The goal is not a perfect backtest. The goal is a fast and honest read on whether the forecast can survive a simple rule.

Decision rule

Each model uses a confidence gate based on the larger of a {{ assumptions.threshold_pct|floatformat:2 }}% floor or its own {{ assumptions.signal_quantile_pct|floatformat:0 }}th-percentile predicted move. That keeps quiet models active and prevents noisy ones from trading every hour.

Execution friction

Each active trade pays {{ assumptions.fee_pct|floatformat:2 }}% in simulated cost. This prevents unrealistic “paper alpha” from overtrading.

My thinking

Good ROI pages on strong finance products usually separate forecasting quality from decision quality. I followed that pattern here: the user sees forecast models above, but this page answers whether those signals actually improve capital and risk once translated into action.
